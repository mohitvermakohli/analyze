Welcome to the Data Processing Project

This project showcases a complete data workflow from raw input to processed output, leveraging Python with Pandas for data manipulation and GitHub Actions for continuous integration and deployment. The goal is to automate the conversion, processing, and publication of data results.

We start with an Excel file (`data.xlsx`), convert it to a more accessible CSV format (`data.csv`), and then use a Python script (`execute.py`) to perform meaningful aggregations. The final processed data is then made available as a JSON file (`result.json`) via GitHub Pages.

The Data Workflow

1. Input Data

The project starts with raw data provided in an Excel format: data.xlsx. This file is converted to data.csv for easier processing.

2. Processing Script

The execute.py Python script, built with Pandas, reads data.csv, performs aggregations (e.g., summing values by category), and outputs structured data.

3. CI/CD with GitHub Actions

Our .github/workflows/ci.yml automates the entire process: linting with Ruff, running execute.py, and publishing the results to GitHub Pages.

This automated pipeline ensures that any changes to the data or processing logic are immediately reflected in the published results, maintaining data freshness and consistency.

Accessing Processed Results

The final processed output, `result.json`, is automatically generated by the CI/CD workflow and published via GitHub Pages. You can view the latest results directly:

Link to results:

https://YOUR_USERNAME.github.io/YOUR_REPOSITORY_NAME/result.json

(Remember to replace `YOUR_USERNAME` and `YOUR_REPOSITORY_NAME` with your actual GitHub details.)

This JSON file contains the aggregated data, ready for consumption by other applications or for direct viewing in your browser.
